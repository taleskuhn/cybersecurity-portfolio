# Case Study 001 — Botium Toys Security Audit

## Executive Summary

This case study evaluates the security posture of Botium Toys through a controls and compliance assessment based on industry best practices.

The objective was to identify existing security controls, evaluate their effectiveness, assess compliance requirements, and recommend improvements that reduce organizational risk.

---

## Organization

**Botium Toys**

A fictional toy retailer used in the Google Cybersecurity Professional Certificate.

---

## Objectives

- Evaluate existing security controls
- Assess compliance requirements
- Identify security gaps
- Recommend improvements based on risk

---

## Frameworks & Standards

- NIST Cybersecurity Framework (CSF)
- PCI DSS
- GDPR
- SOC

---

## Skills Demonstrated

- Risk Assessment
- Security Controls Evaluation
- Compliance Analysis
- Technical Documentation
- Security Recommendations

---

## Key Findings

| Finding | Risk |
|----------|------|
| No IDS | High |
| No Disaster Recovery Plan | Critical |
| Weak Password Policy | Medium |
| Missing Encryption | Critical |

---

## Recommendations

The complete recommendations are available in the attached report.

Highlights include:

- Implement Role-Based Access Control (RBAC)
- Strengthen password policies
- Deploy Intrusion Detection Systems (IDS)
- Encrypt sensitive customer data
- Implement disaster recovery procedures

---

## Lessons Learned

One of the main lessons from this assessment was understanding that the existence of a security control does not necessarily mean it is effective.

A mature security program requires continuous evaluation, improvement, and alignment with organizational risk.
